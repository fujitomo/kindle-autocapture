# Kindle AutoCapture ユーザーマニュアル

PC 版 **Kindle for Windows** を対象に、自動でページ送りしながら画面を連番保存するアプリの操作説明です。

---

## 1. このアプリでできること

- タイトルに「Kindle」を含むウィンドウを一覧から選択
- ウィンドウ全体、またはドラッグで指定した矩形範囲（ROI）をキャプチャ
- 各ページ保存後にキーボード・ホイール・クリックで次ページへ進む
- 知覚ハッシュ（pHash）で「同じ画面が続いた」ときに重複スキップや最終ページの自動停止
- グローバルホットキーで開始・一時停止・停止（アプリが背面でも操作可能な場合あり）
- セッション終了時に連番画像を **1 冊の PDF** に自動まとめ（設定でオン/オフ・ファイル名指定可）

---

## 2. 動作環境

| 項目 | 内容 |
|------|------|
| OS | Windows 10 / 11 |
| 対象アプリ | Kindle for PC（ウィンドウが表示されていること） |
| Python | 3.10 以上（ソースから実行する場合。3.11 推奨） |

管理者権限は必須ではありませんが、**グローバルホットキー**が効かない環境では、管理者として起動すると改善することがあります。

## 3. インストールと起動

### 3.1 ソースから実行

```powershell
cd <本リポジトリのフォルダ>
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python main.py
```

### 3.2 EXE（PyInstaller）

```powershell
pip install pyinstaller
pyinstaller build.spec --clean --noconfirm
```

成果物は `dist` 配下に出力されます（`build.spec` の内容に従います）。

## 4. 画面構成（メインウィンドウ）

| 領域 | 説明 |
|------|------|
| **ターゲットウィンドウ** | キャプチャ対象の Kindle ウィンドウ。**再検索**で一覧を更新。 |
| **保存先フォルダ** | 画像の出力先。**参照…**で変更、**フォルダを開く**でエクスプローラーを開く。 |
| **キャプチャ設定（簡易）** | ページ送り後の待機、最大ページ数、重複スキップ、最終ページ自動停止、実行中の最小化、ページ送り方向の反転。詳細はツールバーの **設定**。 |
| **キャプチャ範囲** | 既定はウィンドウ全体。**範囲を選択…**で画面上にオーバーレイが出るのでドラッグして矩形を指定。**ウィンドウ全体に戻す**でリセット。 |
| **実行状態** | 状態表示、取得枚数、経過時間、**開始 / 一時停止 / 停止** ボタン。 |
| **ログ** | 実行中のメッセージ。**クリア**で表示のみ消去。 |
| **ツールバー「設定」** | タブ分けされた詳細設定ダイアログを開く。 |

---

## 5. 基本の操作手順

1. **Kindle for PC** で対象の本を開き、通常どおり読める状態にする（最大化推奨）。
2. 本アプリで **再検索** し、一覧からその Kindle ウィンドウを選ぶ。
3. 必要なら **保存先** を確認・変更する。
4. 必要なら **範囲を選択…** で本文だけ切り取る（表紙や UI を除きたいとき）。
5. **ページ送り後の待機** を、PC の描画速度に合わせて調整（既定 1.5 秒前後から試す）。
6. **実行中はアプリを最小化（推奨）** をオンにしたまま **▶ 開始** を押す。  
   - 開始時に本アプリが最小化され、Kindle 側にフォーカスが移りやすくなります。
7. 停止するときは **停止**、**Esc**（緊急停止の既定）、または **F9**（開始と兼用で「実行中なら停止」）など。

**最大ページ数** が 0 のときは無制限です（最終ページ検知や手動停止まで続きます）。

---

## 6. グローバルホットキー（既定値）

| 操作 | 既定キー | 動作 |
|------|-----------|------|
| 開始 / 停止 | F9 | 停止中なら **開始** と同じ。実行中・一時停止中なら **停止**。 |
| 一時停止 / 再開 | F10 | 実行中は一時停止、一時停止中は再開。 |
| 緊急停止 | Esc | 停止。 |

キーは **設定 → ホットキー** で変更できます。設定で **OK** したあと、メイン画面に戻るとホットキーは再登録されます。

ホットキーが効かない場合の例:

- 他アプリとキーが競合している
- ゲームモードやセキュリティソフトの制限
- **管理者として** 本アプリを起動してみる

## 7. 詳細設定（設定ダイアログ）

### 7.1 キャプチャ

| 項目 | 意味 |
|------|------|
| ページ送り後の待機 | 次ページの描画が落ち着くまでの秒数。 |
| 初回フォーカス後の遅延 | セッション開始直後の余裕時間。 |
| 保存フォーマット | png / jpeg / webp。 |
| JPEG / WebP 品質 | 各フォーマットの品質。 |
| 重複ページをスキップ | 前ページと画像が似ているとき保存をスキップ。 |
| pHash 重複閾値 | 重複判定の緩さ（ハミング距離。大きいほど「似ている」とみなしやすい）。 |
| 最終ページ検知で自動停止 | オフにすると末尾で止めずに手動停止まで続行。 |
| 最終ページ判定の厳密距離上限 | 「本当に同じページが続いた」とみなす距離の上限。大きすぎると隣接ページで誤停止しやすい。 |
| 最終ページ判定 連続重複回数 | 上記距離以下の「同一扱い」が何回続いたら最終ページとみなすか。 |
| 最終ページ判定までの最低保存枚数 | この枚数未満では最終ページ自動停止しない（誤検知防止）。 |
| 重複スキップ連続の上限（保存なし） | 保存なく重複スキップだけが続いたら **同一画面のまま送り続けた** と判断して停止。ページ送りが Kindle に届いていないときの無限ループ防止。 |
| ウィンドウ内容のみ取得（PrintWindow・重なり除外） | **既定オン**。Kindle の上に Cursor など別ウィンドウが重なっていても、その内容を写さず Kindle の描画だけを取ります。画面が真っ黒になる場合はオフにして従来の画面キャプチャへ。 |

### 7.2 ページ送り

| 項目 | 意味 |
|------|------|
| ページ送り方式 | **key**（キー）または **click**（クリック）。 |
| 送りキー | 例: `pagedown`（既定）。英小文字・区切りは `page_down` も可。 |
| キー送り方式 | 下表参照。 |
| ホイールノッチ数 | ホイール系のみ。**1 推奨**。大きいと 1 操作で複数ページ動くことがある。 |
| ホイール符号 | 環境によって前後が逆のとき **+1** と切り替え。 |
| ページ送り方向を反転 | **Page↔PageUp**、**左右**・**上下**の入れ替えと、ホイール符号の反転をまとめて行う。縦書きや版によって「進む方向」が逆に感じるときに使用。 |
| クリック X/Y オフセット | **click** 方式時、ウィンドウ右端・縦中央からのずれ。 |
| 操作前にウィンドウを前面化 | Kindle を前面にしてから送り操作を行う。 |

**キー送り方式の目安**

| 方式 | 説明 |
|------|------|
| **sendinput** | Page Down 等を低レベル送信。**本アプリ最小化**と **pagedown** の組み合わせが安定しやすい。 |
| **wheel_pyautogui** | マウス位置でホイール。ノッチ数は 1 に。 |
| **wheel_postmessage** | 子ウィンドウへ `WM_MOUSEWHEEL`。 |
| **postmessage** / **pyautogui** | 補助的な手段。 |

`sendinput` で使える主なキー名（内部で仮想キーにマッピングされるもの）の例: `right`, `left`, `pagedown`, `pageup`, `space`, `down` など。反転オプションがオンなら、論理キーと実際に送るキーが入れ替わります。

### 7.3 ホットキー

開始/停止・一時停止・緊急停止のキー文字列（例: `f9`, `ctrl+shift+s`）。空にした項目は既定値に戻ります。

### 7.4 保存

| 項目 | 意味 |
|------|------|
| サブフォルダテンプレ | セッションごとのフォルダ名。`book_{timestamp}` など。 |
| ファイル名テンプレ | 連番ファイル名。`page_{index:04d}` のように `index` を含めます。 |
| 既存ファイル番号から再開する | 同じフォルダに既に `page_0001` などがある場合、続きの番号から保存。 |
| セッション終了時に PDF を自動生成 | オンで、停止時にそのセッション内の連番画像を **1 つの PDF** にします（既定: オン）。 |
| PDF ファイル名 | セッションフォルダ直下に保存する PDF の名前（既定: `book.pdf`）。 |

**PDF について:** 取得枚数が 1 枚以上のときだけ生成します。画像はファイル名に含まれる連番で並べます。ページ数が非常に多いとメモリを多く使う場合があります。不要なら **セッション終了時に PDF を自動生成** をオフにしてください。

## 8. 設定ファイルの場所

- **パス**: `%APPDATA%\KindleAutoCapture\config.json`
- GUI で変更した内容はここに JSON で保存されます。バックアップや他 PC へのコピーに利用できます。

既定の保存先（画像）は、未設定時 **ピクチャフォルダ配下の `KindleAutoCapture`** になります（`Pictures` が無い場合はホーム直下に作成される場合があります）。

---

## 9. 停止時に表示される理由

| 表示 | 意味 |
|------|------|
| ユーザー操作による停止 | 停止ボタン・ホットキーなどで止めた。 |
| 最終ページ検知により停止 | 同じ画面が条件を満たして続いたと判断。 |
| 上限ページ数に到達 | **最大ページ数** に達した。 |
| 同一画面のまま送り続けたため停止 | ページ送りが効いていない可能性。キー方式・最小化・反転を確認。 |
| エラーにより停止 / ディスク空き容量不足 | ログを参照。 |

## 10. トラブルシューティング

| 症状 | 確認すること |
|------|----------------|
| ページが進まない | **実行中はアプリを最小化** をオン。送り方式 **sendinput**、送りキー **pagedown**。Kindle が前面か。 |
| 逆方向にめくれる | **ページ送り方向を反転** を切り替える。ホイールなら **符号** も。 |
| 一気に表紙付近へ戻る | ホイールの **ノッチ数** が 1 か。大きい値は複数ページ分動くことがある。 |
| すぐ「最終ページ」で止まる | **最終ページ判定** の距離・連続回数・**最低保存枚数** を調整。 |
| ホットキーが効かない | キー競合、管理者起動、ログの WARNING を確認。 |
| キャプチャ領域がおかしい | Kindle を最小化していないか。前面化してから再開。 |
| 範囲選択が使いにくい | オーバーレイ表示中は手順に従い、キャンセル時はメイン画面に戻る。 |
| Cursor や他ウィンドウが写り込む | 設定の **ウィンドウ内容のみ取得（PrintWindow）** をオン（既定）。画面の矩形ではなくウィンドウ自身の描画を取得するため、重なりは写りません。真っ黒になる場合はオフにし、IDE は最小化するか Kindle から離す。 |

メイン画面の **ログ** に加え、ファイルは `%APPDATA%\KindleAutoCapture\logs\app_YYYYMMDD.log`（日付ごと）に出力されます。

## 11. 利用上・法的な注意

- 本ツールは **購入済みコンテンツの個人的な範囲での利用** を想定しています。
- 取得画像の**無断再配布・公衆送信**は著作権侵害となる可能性があります。
- Kindle の利用規約・地域の法律を遵守してください。
